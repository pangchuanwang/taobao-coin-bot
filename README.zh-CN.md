# taobao-coin-bot

[English](README.md) | 简体中文

基于 Python + uiautomator2 的安卓 UI 自动化工具，用于学习和研究 Android 自动化技术。通过 ADB 连接安卓手机，自动完成淘宝淘金币日常任务，支持任务自动发现、优先级评分、执行验证和每日报告生成。

> **声明：** 本项目仅供个人学习和研究 Android UI 自动化技术。用户需自行遵守相关平台服务条款。本项目不是商业产品，不对任何结果做保证。

---

## 项目简介

taobao-coin-bot 通过 USB/ADB 连接安卓手机，启动淘宝 App，导航到淘金币任务页面，自动完成任务发现、效率评分、按优先级执行、验证完成状态的完整工作流。

项目展示了 Android UI 自动化的实用技术，包括元素定位、手势模拟、OCR 文字识别、模板匹配、跨 App 导航和验证码处理。

## 功能特性

### 任务发现
- 通过滑动扫描任务页面，从 Android UI 元素中提取任务名称、副标题、奖励和进度
- 维护持久化任务目录，记录每个任务的首次出现和末次出现时间
- 每日检测新增和消失的任务，标记未知任务待后续分析

### 优先级评分
- 每个任务按 `奖励 / 预计耗时` 计算分数
- 需要付款、下单、邀请好友、授权的任务自动跳过
- 历史执行数据（耗时、奖励、成功率）用于优化后续预估

### 任务处理器
- **浏览**：定时滑动浏览商品/活动页面
- **签到领取**：签到和领取奖励
- **搜索浏览**：等待搜索结果页，滑动浏览，返回
- **深度浏览**：商品详情页慢速滑动浏览
- **活动浏览**：促销活动页（仅滑动，不下单）
- **答题**：检测答案选项，点击一个，领取奖励
- **跨 App**：跳转支付宝/闲鱼完成合作任务，然后恢复回到淘宝

### 执行验证
- 每个任务执行后重新扫描任务列表，确认进度是否增加
- 记录成功/失败样本，用于持续学习优化

### 每日报告
- 每次运行生成 JSON + 文本摘要报告
- 跟踪：已完成、已跳过、失败、新增任务；总奖励；总耗时

### 跨 App 导航
- 3 阶段恢复策略，用于从外部 App（支付宝、闲鱼、天猫）返回淘宝
- 阶段 1：按返回键 / 拉回淘宝前台
- 阶段 2：重启淘宝
- 阶段 3：兜底返回任务页

### 验证码处理
- 检测滑动验证码并执行自动滑动手势

## 技术栈

- **语言：** Python 3.10+
- **Android 自动化：** uiautomator2 (3.5.0) — 通过 ADB 驱动手机 UI
- **ADB：** adbutils（随 uiautomator2 内置）
- **OCR：** ddddocr (1.6.1) — 中文 OCR，截图文字识别
- **图像识别：** OpenCV (4.13.0.92) — 多尺度模板匹配，按钮检测
- **许可证：** Apache 2.0

## 项目结构

```
淘金币任务.py            — 主脚本：任务发现、调度、执行、验证

coin_task_handlers.py    — 任务处理器：浏览、签到、答题、跨 App 等

requirements.txt         — Python 依赖
```

## 本地运行

### 环境要求

- Windows 10/11
- Python 3.10+
- 安卓手机 + USB 数据线
- 手机已开启 USB 调试

### 安装

1. 手机进入 **设置 → 关于手机**，连续点击"版本号" 7 次，开启开发者模式
2. 进入 **设置 → 开发者选项**，打开 **USB 调试**
3. 用数据线连接电脑，手机弹窗选择 **允许 USB 调试**
4. 运行安装脚本：

```powershell
.\setup.ps1
```

5. 确认设备连接：

```powershell
.\.venv\Lib\site-packages\adbutils\binaries\adb.exe devices
```

6. 初始化 uiautomator2：

```powershell
.\.venv\Scripts\python.exe -m uiautomator2 init
```

### 每日运行

确保手机已亮屏解锁、淘宝已登录，然后执行：

```powershell
.\run_daily_taobao.ps1
```

日志保存在 `logs/` 目录。

## 使用说明

工具自动完成以下流程：
1. 打开淘宝并导航到淘金币任务页
2. 扫描所有可用任务
3. 对任务评分并排序
4. 按优先级顺序执行任务
5. 验证每个任务的完成状态
6. 生成每日报告

**重要：** 执行期间请勿操作手机——工具通过 ADB 控制 UI。

## 项目亮点

- **装饰器注册机制**：新增任务类型只需装饰一个函数即可
- **历史学习**：任务执行结果（耗时、奖励、成功）持久化存储，用于优化后续预估
- **多信号分类**：11 个优先级类别、5 个跳过类别、10 个标签类型
- **跨 App 恢复策略**：3 阶段超时恢复，从外部 App 安全返回淘宝
- **拟人化操作**：随机化滑动坐标、时长和等待间隔
- **完整报告**：每次运行生成结构化 JSON + 可读文本报告
